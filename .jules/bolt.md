## 2024-03-01 - Optimizing PCA for Sparse Matrices
**Learning:** scikit-learn's `PCA(n_components="some_fraction")` does not natively support sparse matrices because `svd_solver='full'` requires dense conversions. When `AdaptiveWeights` tries to achieve `variability_pct < 1` for sparse data by iteratively incrementing `n_comp` from `1` to `max_comp` and re-running PCA, it implicitly triggers a dense conversion on every loop iteration, leading to massive memory and performance overhead (23s+ for small inputs).
**Action:** Replace iterative PCA incrementing with a single run of `PCA(n_components=max_comp, svd_solver="arpack")` to calculate all principal components at once without iterative dense conversions. Then compute the cumulative explained variance manually to truncate components.

## 2024-05-15 - Vectorizing Group Weight Calculation in Numpy
**Learning:** Using `np.bincount` to compute aggregated metrics over grouped indices can fail via memory explosion if the indices values are large (e.g. they don't map to a continuous small range). A safer and much more efficient way to compute per-group aggregations like squared sum or L2 norm is using `np.argsort` to sort the arrays followed by `np.unique(..., return_index=True)` and `np.add.reduceat(sorted_array, group_starts)`. This vectorizes operations that were previously slow loops, reducing calculation time significantly.
**Action:** When aggregating or applying metrics across groups defined by arbitrary indices, use `argsort` and `reduceat` combination rather than python loops or `np.bincount` to achieve optimal memory-safe performance.
