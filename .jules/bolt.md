## 2024-03-01 - Optimizing PCA for Sparse Matrices
**Learning:** scikit-learn's `PCA(n_components="some_fraction")` does not natively support sparse matrices because `svd_solver='full'` requires dense conversions. When `AdaptiveWeights` tries to achieve `variability_pct < 1` for sparse data by iteratively incrementing `n_comp` from `1` to `max_comp` and re-running PCA, it implicitly triggers a dense conversion on every loop iteration, leading to massive memory and performance overhead (23s+ for small inputs).
**Action:** Replace iterative PCA incrementing with a single run of `PCA(n_components=max_comp, svd_solver="arpack")` to calculate all principal components at once without iterative dense conversions. Then compute the cumulative explained variance manually to truncate components.

## 2024-03-02 - PLS Redundant Fits and CVXPY Parameters
**Learning:** In CVXPY with solvers like CLARABEL, forming the quantile loss objective using residual splitting with explicit non-negative variables (`u - v = residual`) compiles and solves significantly faster than using `cp.abs()` despite graph atom reduction claims. Additionally, using `cp.Parameter` for weights that are not reused throws "Not DPP" warnings and adds a small overhead; directly using numpy arrays is faster for dynamic graphs. Furthermore, computing all PLS components initially in `_wpls_pct` just to count the required `n_comp`, and then refitting from scratch when `n_comp == max_comp` is entirely redundant since the first fit already calculated everything.
**Action:** Removed unnecessary `cp.Parameter` wrappers for `lambda1` and weights across all CVXPY penalty expressions to resolve DPP warnings and speed up canonicalization. Avoided a completely redundant second PLS fit when `n_comp == max_comp`. Kept residual splitting for quantile regression due to better actual benchmarking performance.
