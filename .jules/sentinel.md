## 2024-05-24 - [Insecure Reflection in Weight Technique Dispatch]
**Vulnerability:** The codebase allowed arbitrary attribute execution (reflection) due to `getattr` usage. Specifically, `getattr(self, "_w" + self.weight_technique)` in `skmodels.py` concatenated an attacker-controlled string (`self.weight_technique`) without a restrictive allowlist, potentially permitting execution of unintended methods if a user supplied a maliciously crafted `weight_technique` string.
**Learning:** Even internal attributes and seemingly constrained string values must be strictly validated against an explicit allowlist before being used in dynamic reflection like `getattr` or `eval`. Implicit constraints or reliance on downstream errors are insufficient for security.
**Prevention:** Always implement an explicit allowlist for user-controlled strings that dictate control flow or method dispatch. The validation logic should be executed early (e.g., in `_check_attributes`) before any `getattr` calls.
