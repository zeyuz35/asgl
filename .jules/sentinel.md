## 2024-03-10 - [Fix Insecure Reflection via getattr]
**Vulnerability:** Insecure reflection via `getattr` when dynamically calling methods based on user input. Specifically, the `weight_technique` parameter was passed unvalidated into `getattr(self, "_w" + self.weight_technique)`, which could potentially allow an attacker to invoke unintended private methods if they could control the `weight_technique` string.
**Learning:** Even internal private methods can be invoked if user input is concatenated and passed directly into reflection utilities like `getattr()`. Scikit-learn validation like `check_scalar` doesn't enforce strict domain allowlists by itself unless explicitly implemented.
**Prevention:** Always strictly validate user input against an explicit allowlist of expected values before passing it to reflection utilities like `getattr`, `eval`, or `exec`. Use idiomatic Python type checking and membership tests (`if not isinstance(val, str): ...`, `if val not in ALLOWED_VALUES: ...`).
